# Monad Style & Patterns Reference

## Purpose

This document defines the governed style and patterns for Monad OS / Monad CLI.

Style is not aesthetic.  
Style is governance.

Patterns are not convenience.  
Patterns are invariants.

---

## Style Doctrine

Style must ensure:

```text
clarity
determinism
honesty
explicitness
governance alignment
```

Style is part of the institutional fabric.

---

## Rust Style

### Formatting

```bash
cargo fmt --all
```

### Linting

```bash
cargo clippy --workspace
```

### Patterns

- Prefer explicit over implicit.  
- Prefer deterministic over heuristic.  
- Prefer local-first over remote-first.  
- Prefer honest placeholders over silent stubs.  
- Prefer stable schemas over ad-hoc structures.  

---

## CLI Style

- Commands must be honest.  
- Commands must be explicit.  
- Commands must be deterministic.  
- Commands must be documented.  
- Commands must be governed.  

---

## Documentation Style

- Docs must be complete.  
- Docs must be indexed.  
- Docs must be cross-linked.  
- Docs must be governed.  
- Docs must be evidence-producing.  

---

## Style Invariants

```text
style must reflect governance
patterns must reflect invariants
documentation must reflect policy
```

Style is institutional.
