# Meta Documentation Index

This directory contains cross-cutting terminology and classification documents for Monad.

Meta documentation keeps language stable across product, architecture, governance, data, operations, interoperability, and philosophy documents.

## Files

| File | Purpose |
| ---- | ------- |
| `definitions.md` | Normative definitions for key Monad concepts. |
| `glossary.md` | Alphabetical glossary for common terms. |
| `institutional-terms.md` | Project-specific terms that encode Monad doctrine. |
| `taxonomy.md` | Classification system for artifacts, states, commands, and governance records. |

## Purpose

The meta layer helps ensure that terms such as canonical, generated, local-first, plan-backed, hosted projection, policy finding, waiver, context pack, and lifecycle graph mean the same thing across the repository.

## Rules

- Definitions should be stable and concise.
- Glossary entries may be explanatory and cross-linked.
- Institutional terms should preserve project doctrine.
- Taxonomies should be explicit enough for future machine validation.
- New durable terms should be added here when they appear across multiple docs.

## Maintenance Notes

Update this index when terminology files are added, renamed, or superseded.
